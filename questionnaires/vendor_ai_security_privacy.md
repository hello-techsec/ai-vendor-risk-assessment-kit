# AI Vendor Security & Privacy Questionnaire

## A) Product and use case
1. Describe the AI system use case and intended users.
2. Is the system customer-facing? Does it generate content for end users?
3. Does the system support automated decision-making (e.g., hiring, credit, insurance)?

## B) Data handling
4. What types of data are processed (PII, sensitive data, credentials)?
5. Data retention period and deletion guarantees?
6. Is customer data used for training or fine-tuning? If yes, is opt-out available?

## C) Security controls
7. Authentication and access control (SSO, RBAC, least privilege)?
8. Logging and monitoring (what is logged, retention, customer access)?
9. Vulnerability management (pen tests, bug bounty, patch SLAs)?

## D) Model governance
10. Do you provide model cards, eval results, red-teaming evidence?
11. How do you handle prompt injection / jailbreak / unsafe outputs?
12. Can customers configure safety policies and filters?

## E) Resiliency and incident response
13. Availability targets and DR/BCP posture?
14. Incident response SLAs and breach notification process?
15. Sub-processors and supply chain dependencies?

## F) Legal and compliance
16. Privacy notice and lawful basis support (for customer deployments)?
17. Cross-border transfers and data residency options?
18. Evidence: SOC2 / ISO 27001 / DPIA support / security documentation.
